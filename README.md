# AIProxy

一个给 AI 编程助手用的代理工具桌面应用。当 AI 调用模型接口失败时自动重试，让你的编程任务不会因为网络问题而中断。

## 功能特点

- 🖥️ **图形化界面** - 无需手动配置 .env 文件，可视化操作更直观
- 🔄 **自动重试** - 遇到 429、500 等错误自动重试，避免任务中断
- 📊 **实时监控** - 日志查看、请求统计一目了然
- 📁 **多配置方案** - 支持保存多套配置，一键切换
- 🚀 **系统托盘** - 最小化到托盘，后台运行不干扰工作
- 🔧 **开机自启动** - 可选开机自动启动

## 安装

### Windows

1. 下载最新的安装包（MSI 或 NSIS）
2. 双击安装，按提示完成安装
3. 启动应用即可使用

## 使用方法

### 第一步：配置代理

1. 打开应用，在「仪表盘」页面配置代理参数：
   - **本地端口**: 代理服务监听的端口（默认 3000）
   - **目标接口地址**: 要代理的模型 API 地址
   - **最大重试次数**: 失败后最多重试几次
   - **重试延迟**: 每次重试之间的等待时间
   - **重试状态码**: 哪些 HTTP 状态码需要重试

2. 点击「保存配置」

### 第二步：启动服务

点击「启动服务」按钮，看到状态变为「运行中」即表示代理服务已启动。

### 第三步：配置 AI 编程助手

在你的 AI 编程助手（如 CodeBuddy、Cursor、Continue 等）设置中，将模型接口地址改为代理地址：

```
原来：https://api.xxx.com/v1
改成：http://localhost:3000/v1
```

就这么简单！现在 AI 助手调用模型时，如果遇到错误会自动重试。

## 功能说明

### 仪表盘

- 查看服务状态
- 启动/停止代理服务
- 管理配置方案
- 查看快速统计

### 实时日志

- 查看所有请求日志
- 按级别过滤（INFO/WARN/ERROR）
- 自动滚动到最新日志

### 统计面板

- 总请求数
- 成功/失败请求数
- 重试次数统计
- 成功率可视化图表

## 系统要求

- Windows 10 或更高版本

## 开发

### 环境准备

1. 安装 Rust: https://rustup.rs/
2. 安装 Node.js 18+: https://nodejs.org/

### 本地开发

```bash
# 安装依赖
npm install

# 开发模式运行
npm run tauri dev
```

### 构建发布

```bash
# 构建 Windows 安装包
npm run tauri build
```

构建产物位于 `src-tauri/target/release/bundle/` 目录。

## 技术栈

- **后端**: Rust + Tauri 2.x + axum
- **前端**: Vue 3 + TypeScript + Pinia + SCSS
- **打包**: Tauri 内置打包（MSI/NSIS）
